# Contributing

The Agent Dictionary is for clear, practical explanations of agent terminology.

## Definition Rules

- Define the term in plain English first.
- Keep the first paragraph useful to someone who is not already deep in the
  agent ecosystem.
- Use examples that show what the term means in a real system.
- Link to related entries when the reader probably needs adjacent concepts.
- Cite primary sources when a term is tied to a specific standard, protocol,
  library, benchmark, or paper.
- Mark uncertainty instead of smoothing it over. Some agent terms are contested.

## What To Avoid

- Do not define jargon with more jargon.
- Do not treat marketing labels as settled technical categories.
- Do not imply a system is autonomous, safe, reliable, or agentic without saying
  what it can actually do.
- Do not add terms that are only vendor branding unless they are widely used or
  historically important.

## Entry Checklist

Before opening a change, check that the entry has:

- a one-sentence definition
- a practical example
- at least one common confusion
- related terms
- source links if the definition depends on a specific external reference

Use [templates/term.md](templates/term.md) as the starting point.
