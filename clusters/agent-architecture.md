# Agent Architecture

Agent architecture terms describe how an agent system is organized: who decides
what to do, who calls tools, how work is split up, and where humans remain in
the loop.

## Terms

- [Agent](../terms/agent.md)
- [Scaffold](../terms/scaffold.md)
- [Planner](../terms/planner.md)
- [Orchestrator](../terms/orchestrator.md)
- [Autonomy](../terms/autonomy.md)
- [Human-in-the-Loop](../terms/human-in-the-loop.md)

## Useful Distinction

An LLM is a model. An agent system is the model plus the surrounding machinery:
instructions, tools, memory, state, routing, evaluation, and human control.
